What is Klang?

Klang is a small programming language whose compiler pipeline you can watch and hear. Every time you press Run, your program passes through five real compilation stages, and each stage performs with its own instrument — a clean program builds up like a short symphony, and an error stops the orchestra mid-phrase at exactly the stage that failed.

Is it a real, new language? Yes — Klang is implemented entirely from scratch: its own lexer, recursive-descent parser, scope resolver, semantic checker, and tree-walking interpreter, written in dependency-free JavaScript (see src/). No Python runs anywhere. Klang has its own music-themed keywords — you write motif for a function, ensemble for a class, play to output, when / orwhen / otherwise for conditionals, loop … in scale(...) and sustain for loops. The grammar, token set, scoping rules, and gradual type system are Klang's own, specified in docs/LANGUAGE_DESIGN.md.

The five stages (and their instruments)

Lexersplits source into tokens — triangle synth, one note per token
Parserbuilds the syntax tree — plucked string
Scoperesolves every name (LEGB) — soft pad chord
Semantictype-checks before running — FM bell
Executeruns the program — membrane drum, one hit per output line

The pipeline stops at the first stage that reports an error, and the instruments for the stages after it never play. Try the Semantic error example: the lexer, parser, and scope stages perform, the bell strikes a sour note, and Execute stays silent.

The vocabulary

Klang reads like a score. Each keyword is a performance direction:

motifdefine a reusable phrase (a function)
resolvehand a value back from a motif (return)
playperform / print a value to the console
when · orwhen · otherwiseconditionals (if · elif · else)
sustainkeep looping while a condition holds (while)
loop … in …iterate over a sequence (for-each)
scale(a, b)a run of numbers to loop over (range)
stop · skipleave the loop · jump to the next turn (break · continue)
ensemble · tunedefine a class · its constructor
tuttibind a name in the whole-ensemble (module) scope (global)
restthe empty value / silence (None)
length(s)how many characters are in a string

Numbers, strings, True/False, the operators (+ - * / // % **, comparisons, and / or / not), indentation-defined blocks, and # comments work as you'd expect.

How to use the site

  1. Pick an example from the Example menu, or type your own program in the editor.
  2. Press ▶ Run. Notes pop above each token as the lexer reads it, the Symphony roll shows every stage's notes in its own lane, and the pipeline list on the right checks off stages as they pass.
  3. Output appears in the Console; errors appear in a banner naming the stage, line, and reason.
  4. Use the Sound checkbox to mute, and Reset to restore the example.

A taste of the language

# a motif is a reusable phrase (function), with recursion
motif countdown(n):
    when n == 0:
        play("liftoff")
        resolve
    play(n)
    countdown(n - 1)

countdown(3)

# an ensemble groups behaviour (a class); tune is its constructor
ensemble Counter:
    motif tune(self, start):
        self._n = start
    motif bump(self):
        self._n = self._n + 1

c = Counter(10)
c.bump()
play(c._n)   # 11
  • Types: int, float, bool, string, rest (the empty value); explicit conversions via int(), float(), str(), bool().
  • Control flow: when / orwhen / otherwise, sustain (while), loop … in scale(...) or over a string, stop, skip.
  • Scope: lexical LEGB with closures; tutti writes to module scope; undeclared names are caught before the program runs.
  • Checking: gradual — definite type errors like "a" + True are rejected by the semantic stage; the rest is enforced at runtime with the same messages.

Running Klang from the command line

node klang.js examples/hello.klang      # run a program
node klang.js --stages examples/errors/undeclared.klang
node klang.js --tokens examples/hello.klang
node test.js                            # 32 language tests
